# Deep Research Agents

## 概述

Deep Research Agent 是一类能够自主执行多步搜索、推理和长文写作的 LLM Agent 系统。用户给出一个复杂的研究问题（例如「分析 TikTok 被禁对投资风险的影响」），Agent 自动完成问题分解、多轮网络搜索、信息整合、证据溯源，最终产出一份结构化的深度报告——包含文本分析、可视化图表和可追溯的引用。

这不是简单的一问一答，也不是搜索引擎 + LLM 摘要。Deep Research Agent 需要在一个可能持续数十分钟的自主决策链条中，维持规划的一致性、信息的准确性、和上下文的可控性。

## 核心问题

Deep Research Agent 面对的根本矛盾是**长程推理中的控制衰减**。一条流水线 Plan → Search → Write，每一步的输出是下一步的输入。在没有外部控制机制的情况下，三个问题会逐级放大：

1. **目标漂移**：规划阶段生成的子目标往往过于模糊（「分析市场影响」），缺乏可验证的完成标准。模型在执行中容易偏离原始意图。
2. **上下文腐烂（context rot）**：搜索结果的原始 dump 不断累积，噪声、重复、过时内容挤占有效上下文。模型在长上下文中处理信息的精度随长度下降。
3. **证据断裂**：写作阶段位于流水线末端，承接的是前面所有步骤积累的噪声。报告中的 claim 往往无法追溯到具体来源，幻觉和自相矛盾高发。

## 技术演进

- **线性流水线（2024 以前）**：Plan → Search → Write 的标准范式。依赖基座模型的推理能力硬扛积累误差。代表作：GPT Researcher、OpenDeepResearch。
- **Outline 驱动（2024 中）**：在流水线中加入大纲结构，搜索内容动态更新大纲，提升报告组织性。代表作：STORM、WriteHere、WebWeaver。但大纲本身的质量依赖模型，且上下文管理仍是痛点。
- **控制机制注入（2025）**：RhinoInsight 在流水线中加入两层控制——可验证检查清单（VCM）管行为，证据审计（EAM）管上下文。不换模型，只加控制结构，做到 SOTA。核心发现：对 agent 系统来说，控制结构比底层模型的绝对能力更重要。

## 关键概念

**可验证检查清单（Verifiable Checklist）**：将用户问题拆成可逐项验收的子目标，而非模糊的 section 标题。每个子目标有明确的完成标准（「查到具体数字」「找到 3 个真实案例」），规划产出不再是「写一段关于市场影响的分析」而是「找到 TikTok 在美年营收并计算禁令的直接影响」。critic（人或 LLM）审核清单的完整性和可验证性后，才进入搜索阶段。

**证据审计（Evidence Audit）**：两阶段运作。搜索阶段：将原始搜索结果结构化（来源/时间/置信度/摘要）、挂载到大纲节点、裁剪噪声和重复。写作阶段：critic 对证据按相关性/质量/时效性排序，只将高质量证据绑定到具体段落。每句 claim 可追溯到具体来源。

**状态重建（Workspace / State Reconstruction）**：每一步不保留全部历史上下文，而是重建一个轻量 workspace——仅包含当前子目标、相关压缩记忆、上一步最简信息。这使每步的上下文长度保持恒定，防止 long-horizon 下的性能衰减。

## 洞见

Agent 系统的瓶颈不在模型能力，在于**控制结构**。一个中等模型配上好的检查清单和上下文管理，可以超过一个强大模型在无控制流水线中的表现。引申：设计 Agent 系统时，花在「如何约束和引导模型行为」上的精力，应该大于花在「选哪个模型」上的精力。
