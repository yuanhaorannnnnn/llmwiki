# 深度估计

## 概述

深度估计是从图像中推断场景中每个像素的深度（距离相机多远）的技术。核心挑战在于：单目图像本身没有深度信息，模型必须从视觉线索（纹理梯度、透视、遮挡、阴影等）中"猜测"深度。

## 核心问题

**表示之争：离散网格 vs. 隐式场**

深度估计的核心设计选择是"用什么表示深度"：

- **离散网格表示**（传统）：深度图 = 像素网格，每个格点一个深度值。优点是兼容 CNN/ViT 架构，缺点是分辨率被训练尺寸锁定，上采样丢失高频细节。
- **隐式场表示**（InfiniDepth 路线）：深度 = 连续函数 f(x, y) → d。在任意连续坐标查询深度值。突破分辨率限制，但增加逐点查询开销。
- **有序回归**（Ordinal Regression）：Adabins 等把深度分成 bins，预测每个像素属于哪个 bin。可以输出任意 bins 数量的深度，但 bin 边界是离散的。

**相对深度 vs. 度量深度**

- **相对深度**：预测归一化或按比例缩放的深度图，需要 scale-and-shift 对齐才能与真值比较。泛化性强，但无法直接用于需要真实世界的应用。
- **度量深度**：预测绝对深度值（米）。需要额外信息（如训练时的度量深度数据，或推理时提供稀疏深度点）。精度更高但有域差距。

## 主要流派

### 1. 基础模型路线

- **Depth Anything / DAv2 (2024)**：大规模多源数据训练，离散网格表示。通过数据多样性和大规模预训练提升泛化性。
- **Metric3D (2024)**：度量深度，提出"尺度不变"训练策略。
- **UniDepth (2024)**：通用深度，跨相机模型。

### 2. 扩散模型路线

- **Marigold (2024)**：用 Stable Diffusion 进行深度估计，将深度视为"深度图生成"。质量高但速度慢（~秒/张）。
- **MoGe (2025)**：单目度量深度，点云质量突出但分辨率受限。

### 3. 隐式场路线

- **InfiniDepth (2026)**：把深度建模为神经隐式场，任意分辨率 + 细粒度。用子像素监督突破分辨率天花板。

### 4. 提示增强路线

- **Prompt Depth Anything (2024, ZJU)**：不修改深度估计网络，推理时用 LiDAR 稀疏提示（192×256，噪声大）引导 Depth Anything v2 的特征提取。零初始化卷积层保证通用能力不被破坏。关键洞察：尺度信息不需要密集，只需要存在——稀疏 LiDAR 锚点足以"锚定"整个深度图的绝对尺度。
- **Depth Pro (2025)**：锐利深度边缘 + 边界感知，专注高质量深度边缘。

### 5. 多视角/立体路线

- **DeepMVS**：多视角立体匹配
- **DynamicStereo**：动态场景立体匹配
- **AnyFlow**：光流引导深度

## 关键概念

**"分辨率不是参数，是采样密度"**

传统方法：分辨率是网络架构参数（输出多少个像素）。InfiniDepth：分辨率是推理时选择的采样密度（查询多少个坐标点）。

这类似于 NeRF 中的"分辨率无关渲染"——网络不知道分辨率，只需要输入坐标点。

**"子像素监督 = 老师比学生眼睛更好"**

训练时用 4K 真值监督 512 输入的模型。模型被迫学到"坐标 → 深度"的映射而非"网格 → 深度"的映射。就像老师用显微镜看，学生虽然只用肉眼训练，但是学会了"教的方法"而不是"看的方法"。

**"表面均匀采样 ≠ 网格均匀采样"**

网格均匀采样 = 在图像平面均匀分布查询点 → 远处物体点稀疏，近处密集。表面均匀采样 = 按 3D 表面面积分配查询点 → 点云在物体表面均匀分布。

**"稀疏锚点定尺度"**

Prompt Depth Anything 的发现：绝对尺度信息不需要稠密。192×256 的 LiDAR 点（远低于图像分辨率）足以给整张深度图提供正确的绝对尺度。这类似于"稀疏锚定"——基础模型提供形状，外部传感器提供尺度锚点，两者互补。

## 评估标准

| 指标 | 含义 | 最佳值 |
|------|------|--------|
| RMSE | 均方根误差 | 0 |
| AbsRel | 绝对相对误差 | 0 |
| δ < 1.25 | 预测与真值偏差小于 1.25 倍的比例 | 100% |
| SILog | 尺度不变对数误差 | 0 |

## 洞见

**"表示即能力上限"**——整个 CV 领域反复出现的主题。离散表示方便计算但限制能力，连续表示计算成本高但解锁新能力（分辨率无关、表面均匀采样）。找到表示和计算之间的最佳平衡点是 CV 架构设计的核心。

**"稀疏提示 + 稠密基础模型"**——Prompt Depth Anything 展示了另一种范式：不改变表示方式，而是用外部传感器锚定基础模型。基础模型负责形状/几何，传感器负责尺度/锚点。零初始化融合保证基础模型的能力不被破坏。这种分工可以迁移到很多场景——不懂怎么教模型，但可以给模型配一个"示教器"。
