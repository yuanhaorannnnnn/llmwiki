# Redundancy-based Reliability Estimation

> 安全关键系统（自动驾驶、航空航天、医疗设备）中，判断一个传感器/观测源靠不靠谱，需要知道它犯了多少错；要数清楚错误，需要有标准答案（ground truth）来对照。但真实世界里 ground truth 往往不可得或极其昂贵。基于冗余的可靠性估计利用**多个独立（或弱相关）观测源之间的交叉信息**，在没有 ground truth 的情况下反推每个源的可靠性参数。

## 定义

设有 n 个观测源同时对同一物理量做二元判断（检测到/未检测到，比如"前方是否有车"）。在时刻 t，观测结果为向量 **O** = (o₁, o₂, ..., oₙ)，其中 oᵢ ∈ {0, 1}。

传统方法需要 ground truth G ∈ {0, 1} 才能计算：
- **检测率 POD** = P(Oᵢ = 1 | G = 1) —— 真有车时传感器报出来的概率
- **虚警率 PFA** = P(Oᵢ = 1 | G = 0) —— 没车时传感器误报的概率

**基于冗余的方法**：不把 G 当作已知条件，而是当作**隐藏变量**，通过概率模型从观测数据 **O** 中联合估计 G 的分布和各传感器的 (POD, PFA)。

## 核心方法：Berk et al. (2019) 框架

### 1. 问题设定

三个传感器，8 种状态组合：

```
传感器:   1    2    3
        ┌─────────────┐
        │ 0    0    0 │  全没检测到
        │ 0    0    1 │  只有3检测到
        │ 0    1    0 │  只有2检测到
        │ 0    1    1 │  2和3检测到
        │ 1    0    0 │  只有1检测到
        │ 1    0    1 │  1和3检测到
        │ 1    1    0 │  1和2检测到
        │ 1    1    1 │  全检测到
        └─────────────┘
```

每次观测到一种组合，有两种可能的"幕后真相"：要么前面真有车，要么没车。分别计算两种情况下的概率，按"前方有车的先验概率"加权平均——这就是**似然函数**。

### 2. 似然函数（核心引擎）

$$\mathcal{L}(\boldsymbol{\theta} | \mathbf{O}) = \prod_{t=1}^{T} \sum_{g \in \{0,1\}} P(\mathbf{O}_t | G = g, \boldsymbol{\theta}) \cdot P(G = g)$$

其中 θ 包含所有传感器的 POD、PFA 参数。没有 ground truth 时，把 G 积分掉（边际化），用 EM/MCMC 联合估计 θ 和 G 的分布。

**关键洞察**："没有标准答案"不是死局，而是把标准答案当作**隐变量**，用概率模型反推。

### 3. 误差相关性：高斯 Copula + 单因子近似

传感器不是独立的——雨天里摄像头和 LiDAR 可能一起误报。忽略相关性会严重扭曲估计。

**高斯 Copula**：把"非独立的二元判断"映射回"多元正态分布"，用相关系数描述传感器误差之间的"一起犯错"倾向。

**单因子近似**（Dunnet-Sobel 类矩阵）：
- 传感器 i 和 j 的误差相关性 = λᵢ × λⱼ（因子载荷乘积）
- 把 n(n-1)/2 个相关系数缩减到 2n 个载荷参数（FN 和 FP 各一组）
- 计算量：n 维积分 → 一维积分

### 4. 最少需要多少传感器？

| 场景 | 最少传感器数 | 原因 |
|------|------------|------|
| 误差独立 | 3 | 足够多的方程解出各传感器的 POD/PFA |
| 误差相关 | 5 | 需要额外估计相关性参数 |

### 5. 实验结果

- 1000 万次合成观测下，各传感器的 POD 和 PFA 被正确还原，误差在采样噪声范围内
- 即使误差相关性不完全符合单因子假设，估算结果仍然接近真实值
- 相关性平均平方误差：0.02–0.07

**重要限制**：全部实验为合成数据，无真实传感器数据集验证。

## 关键概念详解

### Ground-truth-free ≠ 不需要标定

论文的明确结论：**有限量的含参考真值的测试仍然是必要的**。

目的不是覆盖所有场景（那不可能），而是：
1. **验证误差相关性模型的选择** —— 单因子近似是否适用于你的传感器组合？
2. **排除尾部事件** —— "所有传感器同时犯错"这类最危险但频率极低的事件，纯冗余方法可能低估

实际工作流：少量标定数据确定模型 → 大规模冗余数据扩充分布估计。

### 似然函数的两面性

| 场景 | 似然函数的含义 |
|------|---------------|
| Ground truth 已知 | P(O \| G, θ) —— 给定真相，传感器这么输出的概率 |
| Ground truth 未知 | Σ_g P(O \| G=g, θ)·P(G=g) —— 把真相的各种可能性枚举并加权平均 |

从"条件概率"到"边际概率"，只是把条件换成了求和。数学上 trivial，工程上解决了"没有标准答案怎么做统计"的问题。

## 可迁移洞见

### "冗余 = 隐式 ground truth"

多个弱相关观测源的一致性信息，本身就是一个概率化的 ground truth。这不是放弃标准答案，而是用冗余信息来**合成**一个。

迁移场景：
- **多 IMU 故障检测**：三个惯性测量单元，用一致性判断哪个在漂移
- **多 LLM 一致性评估**：同一个 prompt 发给 3 个模型，不一致时判断谁更可靠
- **多模型 ensemble**：分类任务中，模型 A 和 B 一致、C 不一致时，给 C 降权
- **众包标注质量估计**：没有 gold standard 时，从多个标注员的交叉判断推断各自准确率

### "单因子近似"的工程智慧

把 n(n-1)/2 个参数降到 2n 个，牺牲了相关性刻画的精度，换取了计算可行性。这是安全工程里的常见权衡：
- 不追求模型完美，追求"足够好且能算完"
- 用实验验证近似误差是否在可接受范围内

类似思路：
- Kalman 滤波假设高斯噪声（不完全真实，但可计算）
- 蒙特卡洛假设随机采样能代表全空间（不完全真实，但可收敛）

## 与其他方法的关系

```
可靠性评估方法谱系
    ├── 有 ground truth
    │     ├── 手工标定（封闭场地摆假人）— 精确但慢
    │     └── 人工标注（人眼看视频）— 贵且覆盖有限
    └── 无 ground truth
          ├── 基于冗余（本文）— 多传感器交叉推断
          ├── 基于一致性（Cranmer 等）— 模型自洽性
          └── 基于对抗（GAN-based）— 生成器/判别器博弈
```

**基于冗余 vs. 基于一致性**：
- 冗余：多个**独立**观测源对**同一**物理量做判断
- 一致性：单个模型对**不同角度/变换**的**同一**输入是否输出一致

冗余方法需要硬件/系统层面的多源部署；一致性方法只需要软件层面的多视角变换。两者互补。
